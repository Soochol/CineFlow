# Blog Writing Reference

> 블로그 유형 + 섹션 구조 + 톤/보이스 + 한국어 블로그 규칙 + 품질 체크리스트

## Blog Post Types (8종)

| 유형 | 영어 | 구조 패턴 | 핵심 특징 | 적합한 콘텐츠 |
|------|------|-----------|-----------|---------------|
| How-To 가이드 | How-To Guide | 단계별 지침 | 전제조건→단계→결과 | 실용 정보, 튜토리얼 |
| 리스티클 | Listicle | 번호 목록 | 항목별 간략 설명, 스캔 가능 | 팁, 도구, 추천 |
| 사례 연구 | Case Study | 문제→해결→결과 | 데이터 기반, 인용 | 성공 사례, 고객 경험 |
| 오피니언 | Opinion/Editorial | 주장→논증→결론 | 개인 관점, 근거 제시 | 업계 분석, 트렌드 전망 |
| 비교 분석 | Comparison | 항목별 비교 | 장단점, 추천 | 제품/서비스/도구 비교 |
| 리뷰 | Review | 경험→평가→결론 | 직접 경험, 평가 기준 | 제품/서비스 리뷰 |
| 필라 가이드 | Pillar/Ultimate Guide | 종합 커버리지 | TOC, 클러스터 링크 | 핵심 주제 완전 정리 |
| 기술 튜토리얼 | Technical Tutorial | 환경 설정→코드→트러블슈팅 | 코드 예시, 재현 가능 | 개발자 대상 가이드 |

### 유형별 구조 상세

#### How-To 가이드
1. 서론: 문제 정의 + 이 가이드의 가치
2. 전제조건/준비물
3. 단계 1~N (번호 매기기 필수)
4. 예상 결과/완료 확인
5. FAQ/트러블슈팅
6. 결론 + CTA

- 권장 분량: 1,500~3,000자 (영) / 3,000~6,000자 (한)
- 핵심 규칙: 단계마다 하나의 액션만, 스크린샷/이미지 권장

#### 리스티클
1. 서론: 왜 이 목록인가
2. 항목 1~N (번호 + 소제목 + 설명)
3. 결론: 핵심 정리 + CTA

- 권장 분량: 1,000~2,500자 (영) / 2,000~5,000자 (한)
- 핵심 규칙: 항목 수를 제목에 포함, 가장 강력한 항목을 처음과 마지막에 배치

#### 사례 연구
1. 서론: 기업/프로젝트 소개
2. 도전 과제 (Problem)
3. 해결 방법 (Solution)
4. 결과와 성과 (Result) — 구체적 수치
5. 핵심 교훈 (Key Takeaways)
6. CTA

- 권장 분량: 1,500~3,000자 (영) / 3,000~6,000자 (한)
- 핵심 규칙: 정량 데이터 필수 (%, 금액, 기간), 인용문 포함

#### 오피니언/에디토리얼
1. 서론: 도발적 주장 또는 핵심 논제
2. 배경/맥락
3. 주장 1~3 (각각 근거 포함)
4. 반론 인정 + 반박
5. 결론: 독자 행동 촉구

- 권장 분량: 1,000~2,000자 (영) / 2,000~4,000자 (한)
- 핵심 규칙: 1인칭 사용 가능, 데이터로 주장 뒷받침, 균형 잡힌 시각

#### 비교 분석
1. 서론: 비교 이유 + 독자 Pain Point
2. 비교 대상 개요
3. 비교 기준별 분석 (테이블 권장)
4. 장단점 요약
5. 결론: 상황별 추천

- 권장 분량: 1,500~3,000자 (영) / 3,000~6,000자 (한)
- 핵심 규칙: 비교 테이블 필수, 공정한 평가, 사용 시나리오별 추천

#### 리뷰
1. 서론: 제품/서비스 개요 + 핵심 결론 미리보기
2. 직접 경험 서술
3. 평가 기준별 분석 (기능, 사용성, 가격 등)
4. 장단점 요약
5. 점수/평점
6. 결론: 누구에게 추천하는가

- 권장 분량: 1,500~2,500자 (영) / 3,000~5,000자 (한)
- 핵심 규칙: 직접 경험 기반 (E-E-A-T Experience), 스크린샷/사진 권장

#### 필라/얼티밋 가이드
1. 서론: 주제 개요 + 가이드 가치
2. 목차 (Table of Contents)
3. 섹션 1~N (각각 H2, 하위 H3)
4. FAQ
5. 결론: 핵심 요약 + 관련 글 링크

- 권장 분량: 3,000~5,000자 (영) / 6,000~10,000자 (한)
- 핵심 규칙: TOC 필수, 클러스터 글 내부 링크, 정기 업데이트 날짜 표기

#### 기술 튜토리얼
1. 서론: 무엇을 만들/배울 것인가
2. 사전 요구사항 (환경, 버전, 설치)
3. 단계별 구현 (코드 블록 포함)
4. 실행 결과 확인
5. 트러블슈팅/FAQ
6. 다음 단계/심화 학습

- 권장 분량: 2,000~4,000자 (영) / 4,000~8,000자 (한)
- 핵심 규칙: 코드 블록에 언어 명시, 복사 가능한 완전한 코드, 버전 명시

## Section Structure Rules

### Introduction (Hook)

| 기법 | 영어 | 효과 | 예시 |
|------|------|------|------|
| 충격 통계 | Surprising Statistic | 데이터로 주의 환기 | "기업의 73%가 AI를 도입했지만, 실제 ROI를 측정하는 곳은 12%에 불과하다" |
| 공감 질문 | Relatable Question | 독자 공감 유도 | "블로그 글을 쓸 때마다 3시간 이상 걸리진 않나요?" |
| 과감한 주장 | Bold Statement | 호기심 유발 | "SEO는 죽었다. 적어도 우리가 알던 방식의 SEO는" |
| 고통점 직시 | Pain Point | 문제 직접 언급 | "매달 10개의 블로그 글을 쓰면서도 트래픽이 늘지 않는다면" |
| 스토리텔링 | Storytelling | 감정 연결 | "작년 이맘때, 우리 팀은 블로그 트래픽이 바닥을 쳤다" |
| HAPS | Hook-Agitation-Promise-Solution | 복합 기법 | Hook→문제 심화→약속→해결책 순서 |

**규칙**:
- 영문: 150~200 단어
- 한국어: 300~400자
- 핵심 키워드를 첫 100단어(영) / 200자(한) 내에 포함
- 글의 가치 제안(Promise)을 명확히 전달
- "이번 글에서 알아보겠습니다" 같은 약한 서론 지양

### Body Sections

**헤딩 계층 규칙**:

| 레벨 | 용도 | 규칙 |
|------|------|------|
| H1 | 글 제목 | 1개만 사용 |
| H2 | 주요 섹션 | 3~8개, 키워드 포함 |
| H3 | 하위 섹션 | 필요 시, H2당 2~4개 |
| H4 | 세부 항목 | 최소화 (깊이 3 이상 지양) |

**문단 규칙**:
- 2~3문장 / 문단 (가독성 최적)
- 한 문단 = 한 아이디어
- 전환어(Transition)로 문단 연결

**전환어 예시**:

| 관계 | 한국어 | 영어 |
|------|--------|------|
| 추가 | 또한, 더불어, 나아가 | Additionally, Furthermore, Moreover |
| 대조 | 하지만, 반면에, 그럼에도 | However, On the other hand, Nevertheless |
| 인과 | 따라서, 결과적으로, 그 때문에 | Therefore, As a result, Consequently |
| 예시 | 예를 들어, 구체적으로, 실제로 | For example, Specifically, In practice |
| 요약 | 요약하면, 결국, 핵심은 | In summary, Ultimately, The key point is |

**콘텐츠 강화 요소**:

| 요소 | 영어 | 효과 | 사용 빈도 |
|------|------|------|-----------|
| 통계/데이터 | Statistics/Data | 신뢰도 향상 | 섹션당 1~2개 |
| 인용 | Blockquote | 권위 부여 | 필요 시 |
| 예시 | Example | 이해도 제고 | 핵심 포인트마다 |
| 이미지 제안 | Image placeholder | 시각화 | 500자당 1개 |
| 목록 | Bullet/Numbered list | 스캔성 | 연속 3항목 이상 시 |
| 콜아웃 박스 | Callout/Tip box | 강조 | 주요 인사이트 |

### Conclusion & CTA

**CTA 유형**:

| CTA 유형 | 영어 | 적합한 블로그 유형 | 예시 |
|----------|------|-------------------|------|
| 뉴스레터 구독 | Newsletter signup | 정보형 (How-To, Pillar) | "매주 AI 트렌드를 받아보세요" |
| 관련 글 링크 | Related post link | 시리즈/필라 | "다음 글: AI 도구 비교 분석" |
| 댓글 유도 | Comment prompt | 오피니언 | "당신의 경험은 어떤가요?" |
| 제품 체험 | Product trial | 비교/리뷰 | "14일 무료 체험 시작하기" |
| 자료 다운로드 | Download resource | How-To/Tutorial | "체크리스트 PDF 다운로드" |
| 소셜 공유 | Social share | 바이럴 콘텐츠 | "이 글이 도움됐다면 공유해 주세요" |

**결론 규칙**:
- 본문 핵심 3줄 요약
- 독자의 다음 행동을 명확히 명시
- 영문: 100~150 단어 / 한국어: 200~300자
- 새로운 정보 추가 금지 (본문 요약만)

## Tone & Voice Guide

| 톤 | 영어 | 특징 | 적합 유형 | 언어 수준 |
|----|------|------|-----------|-----------|
| 전문적 | Professional/Authoritative | 데이터 기반, 3인칭, 객관적 | Case Study, Pillar | 고급 |
| 대화체 | Conversational/Friendly | 2인칭(너/여러분), 친근, 유머 | Listicle, Review | 중급 |
| 교육적 | Educational/Informative | 설명적, 단계적, 중립 | How-To, Tutorial | 중급 |
| 설득적 | Persuasive/Compelling | 논증적, 감정 호소, CTA 강조 | Opinion, Comparison | 중~고급 |
| 서사적 | Storytelling/Narrative | 1인칭, 일화 중심, 감정적 | Case Study, Opinion | 중급 |
| 기술적 | Technical/Precise | 전문 용어, 코드 포함, 정확 | Technical Tutorial | 고급 |

**톤 일관성 규칙**:
- 하나의 글에서 하나의 톤을 유지
- 예외: 서론에서 스토리텔링 후 본론에서 교육적 톤으로 전환은 허용
- 한국어: 경어체(합니다/해요/한다) 혼용 금지

## Weak vs Strong

| 요소 | 약한 표현 | 강한 표현 |
|------|-----------|-----------|
| Hook | "이번 글에서 알아보겠습니다" | "매일 3시간을 낭비하고 있다면, 이 방법이 답입니다" |
| Heading | "AI에 대해" | "2026년 AI가 마케팅을 바꾸는 5가지 방식" |
| Body | "AI는 중요합니다" | "McKinsey 보고서에 따르면 AI 도입 기업의 매출은 평균 23% 증가했다" |
| CTA | "댓글 남겨주세요" | "당신의 팀에서 AI를 어떻게 활용하고 있나요? 경험을 공유해 주세요" |
| Transition | (없음) | "이 원칙을 실제로 적용한 사례를 살펴보겠습니다" |
| 데이터 | "많은 기업이 사용한다" | "Fortune 500 기업의 78%가 2025년에 AI 예산을 증액했다 (Gartner, 2025)" |
| 결론 | "이상으로 마치겠습니다" | "오늘 소개한 3가지 전략 중 하나만 적용해도 블로그 트래픽이 달라집니다" |

## Korean Blog Specifics (한국어 블로그)

### Naver SEO Rules

| 항목 | 규칙 | 예시 |
|------|------|------|
| 제목 | 핵심 키워드 앞배치, 30자 이내 | "AI 마케팅 자동화 완벽 가이드 2026" |
| 소제목 | H2에 관련 키워드 포함 | "## AI 마케팅 도구 추천 TOP 5" |
| 본문 | 표절 없는 심층 콘텐츠, 2,000자 이상 | 직접 경험 + 데이터 기반 |
| 키워드 | 롱테일 키워드, 자연스러운 삽입 | "AI 마케팅 자동화 도구 추천" |
| 이미지 | ALT 텍스트에 키워드, 오리지널 우대 | alt="AI 마케팅 시장 규모 그래프" |
| 태그 | 관련 해시태그 5~10개 | #AI마케팅 #자동화 #2026트렌드 |

### Google Korea SEO Rules

| 항목 | 규칙 |
|------|------|
| Featured Snippet | 질문+직접 답변 구조 (첫 1~2문장에 답변) |
| 구조화 데이터 | Article, FAQ, HowTo 스키마 적용 |
| 모바일 최적화 | 반응형 레이아웃, Core Web Vitals |
| 내부 링크 | 관련 글 상호 연결 (Hub-spoke) |

### Korean Writing Style Rules

| 규칙 | 설명 | 예시 |
|------|------|------|
| 경어체 일관성 | 합니다체/해요체/한다체 중 택 1, 혼용 금지 | 합니다체: "~입니다. ~합니다." |
| 영어 전문 용어 | 첫 등장 시 영어(한국어) 병기, 이후 한국어 | "SEO(검색엔진최적화)를... 이후 SEO를..." |
| 문장 길이 | 40자 이내 권장 | 긴 문장은 두 문장으로 분리 |
| 문단 구분 | 3~4문장마다 | 모바일 가독성 고려 |
| 접속사 사용 | 문장 시작에 접속사 활용 | "또한", "하지만", "따라서" |
| 한자어 비율 | 30% 이하 | 쉬운 우리말 우선 사용 |

## Pre-Writing Checklist

- [ ] 블로그 유형이 결정되었는가 (8종 중 택 1)
- [ ] 타겟 독자와 그들의 핵심 Pain Point가 파악되었는가
- [ ] 핵심 키워드(Primary)와 보조 키워드(Secondary)가 정의되었는가
- [ ] Hook 기법이 선택되었는가 (6종 중 택 1)
- [ ] H2/H3 구조가 설계되었는가
- [ ] 각 섹션의 핵심 포인트가 정리되었는가
- [ ] 톤앤매너가 결정되었는가 (경어체 포함)
- [ ] CTA 유형이 선택되었는가 (6종 중 택 1)
- [ ] 참조할 데이터/통계/사례가 준비되었는가

## Post-Writing Checklist

- [ ] Hook(서론)이 150~200단어(영) / 300~400자(한) 범위이며 핵심 키워드를 포함하는가
- [ ] 선택한 블로그 유형의 구조 패턴을 따르고 있는가
- [ ] 문단이 2~3문장(영) / 3~4문장(한) 이내인가
- [ ] 전환어(Transition)로 문단 간 연결이 자연스러운가
- [ ] H2/H3 소제목에 키워드가 포함되었는가
- [ ] 콘텐츠 강화 요소(통계, 인용, 목록, 이미지 제안)가 적절히 배치되었는가
- [ ] 결론이 본문 핵심 3줄 요약 + CTA를 포함하는가
- [ ] 경어체(합니다/해요/한다)가 전체에서 일관되는가
- [ ] 권장 분량 범위 내인가 (유형별 분량 참조)
- [ ] "이번 글에서 알아보겠습니다" 같은 약한 표현이 없는가

## Notes

- 블로그 유형에 따라 섹션 구조와 권장 분량이 달라진다
- 한국어 블로그는 Naver SEO와 Google SEO를 동시에 고려한다
- Hook은 블로그의 성패를 결정하는 핵심 요소이다 — 가장 많은 시간을 투자한다
- SEO 상세 규칙은 `blog-seo` 레퍼런스를 참조한다
- 가독성과 SEO가 상충할 때 가독성을 우선한다
