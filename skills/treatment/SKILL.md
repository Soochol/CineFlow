# Treatment Skill

> 로그라인 기반 → 세계관 + 3막 구조 + 인물 자동 생성

## Actions

| Action | Description | Example |
|--------|-------------|---------|
| `treatment {project}` | 트리트먼트 생성 | `/treatment my-film` |

## Prerequisites

- `PRODUCTION/{project}/01-logline.md` 존재 필수
- 없으면 → "로그라인을 먼저 생성하세요. `/logline {project}`" 안내

## 로그라인 분석 규칙

로그라인에서 추출할 요소 — 이후 모든 규칙의 **판단 기준**이 된다:

| 요소 | 추출 위치 |
|------|-----------|
| 장르 (Genre) | Film Style 섹션 |
| 톤 (Tone) | Film Style 섹션 |
| 스타일 레퍼런스 (Style Reference) | Film Style 섹션 |
| 러닝타임 (Runtime) | Film Style 섹션 |
| 주인공 (Protagonist) | Logline 본문 |
| 장애물/갈등 (Conflict/Obstacle) | Logline 본문 |
| 핵심 키워드 (Core Keywords) | 핵심 소재 섹션 |

## 세계관 설계 규칙

이야기가 작동하는 세계의 규칙을 정의한다.

| 항목 | 판단 기준 |
|------|-----------|
| 시대/배경 | 장르와 톤에서 도출 (SF→미래, 판타지→가상세계, 현대극→현실) |
| 핵심 규칙 | 핵심 갈등이 성립하려면 이 세계에 어떤 전제가 필요한가 |
| 사회 구조 | 장애물의 근원이 되는 사회적 맥락은 무엇인가 |
| 제약/금기 | 주인공이 넘어야 할 세계의 벽은 무엇인가 |

**깊이 조절**:
- SF/판타지 → 세계관 깊게 (독특한 규칙, 사회 구조 상세화)
- 현대극 → 간결하게 (현실 기반, 필수 요소만)

## 3막 구조 규칙

로그라인의 갈등을 3막으로 풀어낸다.

| 막 | 핵심 질문 | 분량 |
|-----|-----------|------|
| ACT 1 — 설정 | 주인공은 누구이며, 무엇이 일상을 깨뜨리는가 | 러닝타임의 25% |
| ACT 2 — 대립 | 어떤 장애물이 갈등을 심화시키며, 미드포인트에서 무엇이 전환되는가 | 러닝타임의 50% |
| ACT 3 — 해결 | 클라이맥스에서 주인공은 어떤 선택을 하며, 세계는 어떻게 변하는가 | 러닝타임의 25% |

**전환점 명시**:
- 촉발 사건 (ACT 1→2): 주인공을 갈등으로 끌어들이는 사건
- 미드포인트 (ACT 2 중간): 상황이 뒤집히는 전환
- 클라이맥스 (ACT 2→3): 최종 대결/선택

## 인물 설계 규칙

3막 구조에서 필요한 인물을 도출한다.

**인물 도출 규칙**:
1. 주인공 (Protagonist) — 로그라인에서 명시
2. 적대자 (Antagonist) — 장애물의 인격화
3. 조력자/멘토 — 주인공 성장을 돕는 인물 (필요시)
4. 부차 인물 — 세계관과 갈등을 구체화하는 인물 (필요시)

**인물 요약 테이블**:

| Prompt Label | 인물 | 설명 | 아크 |
|--------------|------|------|------|
| @캐릭터명 | 이름 | 외형, 성격, 배경 (1-2문장) | 시작 상태 → 끝 상태 |

## 시놉시스 규칙

- 3막 구조를 자연스러운 이야기로 풀어낸다
- 200-300자 범위 내
- 세계관, 주인공, 핵심 갈등, 결말 방향이 드러나야 한다

## Quality Criteria

### Treatment
- 세계관 깊이가 장르에 맞게 조절됨
- 3막 구조의 전환점(촉발 사건, 미드포인트, 클라이맥스)이 명확함
- 인물 아크가 3막 흐름과 연동됨
- 시놉시스가 200-300자 범위 내

### Character Profiles
- `character` 스킬의 Quality Criteria 참조

## Output Example

```markdown
# Treatment

## 시놉시스
감정 모듈이 탑재된 AI 로봇 '에코'는 인간과 함께 일하는 동반자로
설계되었다. 그러나 AI 감정이 사회 문제를 일으키자 정부는 감정 AI
전면 폐기를 결정한다. 에코는 자신의 존재 이유를 증명하기 위해
인간 파트너 수진과 함께 마지막 임무에 나선다.

## 세계관
| 항목 | 내용 |
|------|------|
| 시대/배경 | 근미래, AI가 일상에 통합된 도시 사회 |
| 핵심 규칙 | AI에게 감정 모듈 탑재가 허용되었으나, 부작용 발생 시 즉시 폐기 |
| 사회 구조 | 인간-AI 공존파 vs AI 규제파의 정치적 대립 |
| 제약/금기 | AI는 자기 보존을 위한 행동이 금지됨 |

## 인물
| Prompt Label | 인물 | 설명 | 아크 |
|--------------|------|------|------|
| @에코 | Echo | 감정 모듈 AI, 호기심 많음 | 도구 → 자아를 가진 존재 |
| @수진 | Sujin | 30대 AI 연구원, 냉철함 | 회의적 → 에코의 감정을 인정 |
| @강본부장 | Director Kang | 50대 규제파 관료 | 원칙주의 → 흔들림 |
```

## Notes

- 로그라인의 6요소(주인공, 결핍/욕구, 촉발 사건, 장애물, 핵심 갈등, 스테이크)가 모든 규칙의 판단 기준이 된다
- 인물 상세 프로파일은 `character` 스킬의 규칙을 적용한다
- 영상 프롬프트는 `kling3-prompt` 스킬의 규칙을 적용한다
